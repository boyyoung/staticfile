  Form Bean Definitions … フォームの定義  
  Action Mapping Definitions … アクションの定義  
   
   
   
   
   
   
   
   
   
   
   
   
   
   
   
   
   
                                                                   
                                                                   
  ● フォームの定義  
  画面項目に対応する内容を<form-property>に定義する。  
 
画面項目とform-propertyを一致させることで、自動的にFormクラスのインスタンスが生成され、画面情報が格納される。  
   
   
   
  ←  
   
   
   
   
  <form-bean>  
  name … フォーム名（jsp名 + "_Form"）  
  type … 「org.apache.struts.validator.DynaValidatorForm」固定  
   
  <form-property>  
  name … 画面項目名（=<input>のname）  
  type … データ型（配列指定可能）  
   
   
   
   
   
   
   
   
   
   
   
   
   
   
   
   
   
   
                                                                   
                                                                   
  ● アクションの定義  
  ボタン名（イベントID）、使用フォーム、権限、アクションBean、画面遷移の関連付けを行う。  
 
 
   
   
   
   
   
   
   
   
   
  <action>  
  path … イベントID（xx.doのxxの部分）  
  type … 「com.property.action.GenericAction」固定  
  className … 「com.property.action.BuildActionMapping」固定  
  name … フォーム名（<form-bean>のname）  
  scope … フォームBeanを保存するスコープ  
  input … validateが「true」の場合、Validationチェックエラー発生時の遷移先画面を指定  
  validate … Validationチェックを行うか否かを指定  
   
  <set-property>  
  property "permission" ： valueに権限を設定する（詳細は権限シート参照）  
  "entity" ： 対象Beanを設定する  
  ＊ Bean名 … 完全修飾名で指定  
  ＊ form.*=* … すべての入力項目を意味する  
   
   
  <forward>  
  name … "success"固定  
  path … 正常終了時の遷移先画面  
  異常終了の場合は、「PrException」を投げることで「GenericAction」からエラー画面に遷移  
   
   
   
   
                                                                   
                                                                   
  ● ブックマーク機能  
  キャンセルボタンなどによる画面遷移を制御する。  
  <set-property property="bookmark" value="xxxxx"/>  
  value "save" ： URLとパラメータを記録  
  "return" ： saveで記録したページに戻る  
  "save_parameter" ： パラメータのみを記録  
  "return_parameter" ： save+"save_parameter"に戻る  
   
  −例−  
  @ メニューから画面Ａを表示  
  <action path="/aa_lnk" ... >  
  <set-property property="bookmark" value="save"/>  
  </action>  
  URL＋パラメータ(この場合は、パラメータがないのでURLのみ)が記録される。  
   
  A 画面Ａで検索条件を入力して検索ボタンをクリック  
  <action path="/search_btn" ... >  
  <set-property property="bookmark" value="save_parameter"/>  
  </action>  
  パラメータ(検索条件)が記録される。  
   
  B 検索結果  
  検索結果として、画面Ｂが表示される。  
   
  C 画面Ｂからｎ画面を遷移し、更新後の更新確認画面「ＯＫ」ボタンのクリック  
  <action path="/update_btn" ... >  
  <set-property property="entity"  
  value="result:com.property.app.etc.PrMoCommon_update_msg:const.return_parameter=event,const.-3017=msg_cd"/>  
      <forward name="success" path="/jsp/Success_MsgDisplay.jsp"/>  
  </action>  
  「const.return_parameter=event」として、bookmarkに"return_parameter"が設定してあるイベントを起動する。  
  これで、aa_lnk.do＋パラメータ(検索条件)＋return_parameter=trueがリクエストされる。  
   
  D 画面Ｂに戻る  
  リクエストに「return_parameter=true」があれば、自動的に検索処理を行うようにする(JavaScriptなどを利用)。  
  また画面Ｂの入力項目には「<html:text ... fromrequest="true" />」とする。  
      fromrequest="true"であればリクエストから優先的に情報を値をセットする。
